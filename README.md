# Weather Multi-Agent Project

Проект демонстрирует многошаговую генерацию финального ответа
с использованием нескольких LLM-вызовов (multi-step reasoning)
и внешнего API погоды.

Пользователь вводит город и свои предпочтения, после чего система
последовательно запускает несколько агентов, каждый из которых
отвечает за отдельную часть рекомендации вечернего y8i68iосуга.

## Основная идея проекта

Проект реализует подход, при котором итоговый ответ формируется
не за один запрос к языковой модели, а через цепочку взаимосвязанных шагов:

1. Получение данных о погоде из внешнего API OpenWeather
2. Предложение вариантов досуга на основе погоды (Activity Agent)
3. Подбор еды с учётом выбранной активности и предпочтений пользователя (Food Agent)
4. Подбор подходящих мест в городе (Location Agent)
5. Рекомендации по одежде (Outfit Agent)

Каждый шаг реализован в виде отдельного агента и отдельного вызова LLM.
Каждый последующий шаг использует результат предыдущего, реализуя multi-step reasoning.

## Архитектура проекта

Проект построен по многоагентной архитектуре.

### Агенты

- **Weather Agent**
  - Получает погодные данные через OpenWeather API
  - Преобразует данные в человекочитаемое текстовое описание

- **Activity Agent**
  - Анализирует погодные условия
  - Предлагает варианты вечернего досуга
  - Использует LLM через Groq API

- **Food Agent**
  - Подбирает еду для выбранной активности
  - Учитывает предпочтения пользователя (например, нелюбимые продукты)
  - Запрашивает LLM через Groq API

- **Location Agent**
  - Предлагает подходящие места в выбранном городе
  - Использует LLM для генерации рекомендаций

- **Outfit Agent**
  - Даёт рекомендации по одежде
  - Учитывает погоду и формат активности
  - Использует LLM через Groq API

## Многошаговая генерация (Multi-step LLM)

Финальный ответ формируется через **последовательность LLM-вызовов**:

Пример работы:

1. Weather Agent получает прогноз погоды.
2. Activity Agent предлагает возможные вечерние активности на основе погоды.
3. Food Agent рекомендует, что взять с собой или приготовить, учитывая предпочтения пользователя.
4. Location Agent предлагает подходящие места для выбранной активности.
5. Outfit Agent даёт рекомендации по одежде в соответствии с погодой и активностью.

Каждый шаг использует результат предыдущего, создавая цепочку логических рассуждений.
