# Weather Multi-Agent Project

Проект демонстрирует многошаговую генерацию финального ответа
с использованием нескольких LLM-вызовов (multi-step reasoning)
и внешнего API погоды.

Пользователь вводит город и свои предпочтения, после чего система
последовательно запускает несколько агентов, каждый из которых
отвечает за отдельную часть рекомендации вечернего досуга.

---

## Основная идея проекта

Проект реализует подход, при котором итоговый ответ формируется
не за один запрос к языковой модели, а через цепочку взаимосвязанных шагов:

1. Получение данных о погоде из внешнего API
2. Предложение вариантов досуга на основе погоды
3. Подбор еды с учётом выбранной активности и предпочтений пользователя
4. Подбор подходящих мест в городе
5. Рекомендации по одежде

Каждый шаг реализован в виде отдельного агента и отдельного вызова LLM.

---

## Архитектура проекта

Проект построен по многоагентной архитектуре.

### Агенты

- **Weather Agent**
  - Получает погодные данные через OpenWeather API
  - Преобразует данные в текстовое описание

- **Activity Agent**
  - Анализирует погодные условия
  - Предлагает варианты вечернего досуга

- **Food Agent**
  - Подбирает еду для выбранной активности
  - Учитывает предпочтения пользователя (например, нелюбимые продукты)

- **Location Agent**
  - Предлагает подходящие места в выбранном городе

- **Outfit Agent**
  - Даёт рекомендации по одежде
  - Учитывает погоду и формат активности
